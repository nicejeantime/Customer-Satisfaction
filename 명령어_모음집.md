# 명령어 모음집 (비개발자용)

기획자分이라 터미널 명령어가 낯설 수 있어서, "무엇을, 왜, 어떻게" 위주로 정리했습니다.
대부분은 저(Claude)에게 그냥 말로 시키시면 되고, **딱 2가지(GitHub 로그인, Vercel 로그인)만** 본인이 직접 눌러야 합니다 — 계정 인증이라 대신할 수 없는 부분입니다.

터미널 여는 법: `Cmd + Space` → "터미널" 입력 → Enter

---

## 1. GitHub에 올리기 (최초 1회, 지금 필요한 단계)

로컬에는 이미 커밋 완료해뒀습니다. 아래 명령어를 터미널에 붙여넣고 Enter만 누르면 됩니다.

```bash
cd "/Users/luna.lovegood/Desktop/AI Native/Customer Satisfaction "
git push -u origin main
```

- 실행하면 GitHub 로그인 창이 뜨거나, 사용자명/비밀번호를 물어봅니다.
- **비밀번호 자리에는 GitHub 비밀번호가 아니라 "토큰"을 입력해야 합니다.**
  - 토큰 만드는 곳: https://github.com/settings/tokens → "Generate new token" → 권한(scope)에서 `repo` 체크 → 생성 → 나오는 문자열을 복사해서 비밀번호 자리에 붙여넣기
- 한 번 성공하면 이후엔 이 과정을 다시 안 거쳐도 됩니다 (컴퓨터가 기억함).

## 2. 앞으로 코드가 바뀔 때마다 GitHub에 반영하기

**직접 안 하셔도 됩니다.** 저한테 "깃헙에 올려줘" 또는 "커밋하고 푸시해줘"라고 말씀하시면 아래 과정을 제가 대신 실행합니다.

참고로 뒤에서 일어나는 일:
```bash
git add -A                    # 바뀐 파일 전부 담기
git commit -m "변경 내용 요약"   # 스냅샷 저장
git push                       # GitHub에 반영
```

## 3. 온라인 배포 (Vercel) — 계정 로그인이 필요해서 직접 진행

Vercel은 "이 폴더를 인터넷에 있는 누구나 볼 수 있는 주소로 띄워주는" 무료 서비스입니다. 계정 연결은 보안상 본인이 직접 눌러야 해서, 아래 단계만 브라우저에서 클릭해주시면 됩니다 (터미널 명령어 없음).

1. https://vercel.com 접속 → "Continue with GitHub"로 로그인 (방금 만든 GitHub 계정으로)
2. "Add New..." → "Project" 클릭
3. `nicejeantime/Ray` 저장소 선택 → "Import"
4. 설정 화면에서 **"Root Directory"를 `webapp`으로 변경** (마법사 파일이 있는 폴더)
5. "Deploy" 클릭
6. 배포 완료되면 `https://ray-xxxx.vercel.app` 같은 주소가 나옵니다 — 이게 다른 사람도 접속 가능한 마법사 URL입니다.

이 단계까지 마치시면 저한테 "배포 끝났어" 또는 생성된 URL을 알려주세요. 필요하면 그다음 확인/연결 작업은 제가 이어서 하겠습니다.
