## 2. 구글 설문(실제 응답 페이지) + 대시보드 처음 배포하기

고객사 담당자가 점수를 입력하는 설문 페이지와, 사내에서 결과를 보는 대시보드는 **같은 프로젝트, 다른 배포**로 나뉩니다. **구글 계정 로그인이 필요해서 직접 진행**해야 합니다.

1. [script.google.com](https://script.google.com) 접속 → "새 프로젝트"
2. `webapp/gsurvey/Code.gs` 내용을 복사해서 붙여넣기
3. 좌측 "+" → "HTML" → 파일명을 정확히 `Index` 로 지정 → `webapp/gsurvey/Index.html` 내용 붙여넣기
4. 좌측 "+" → "HTML" → 파일명을 정확히 `Dashboard` 로 지정 → `webapp/gsurvey/Dashboard.html` 내용 붙여넣기
5. `Code.gs` 상단의 `DASHBOARD_DOMAIN` 값을 실제 회사 구글 워크스페이스 도메인으로 수정 (예: `dktechin.com`)
6. **배포 A (설문용)**: "배포" → "새 배포" → 유형: **웹 앱** → 실행 계정: **나** → 액세스 권한: **링크가 있는 모든 사용자** → 배포 → 나오는 URL을 "설문 URL"로 보관
7. **배포 B (대시보드용)**: "배포" → "새 배포" (한 번 더) → 액세스 권한: **"○○○(회사 도메인) 내 모든 사용자"** → 배포 → 나오는 URL을 "대시보드 URL"로 보관

설문 URL은 마법사의 "URL 생성" 단계에서 필요하고, 대시보드 URL은 4번에서 씁니다.

상세 가이드: [webapp/gsurvey/설정가이드.md](webapp/gsurvey/설정가이드.md)

## 4. 대시보드로 고객사별 결과 보기

1. 위 2번에서 만든 **대시보드 URL** 뒤에 `?view=dashboard`를 붙여서 접속
   ```
   {대시보드용 배포 URL}?view=dashboard
   ```
2. 회사 구글 계정으로 로그인되어 있어야 보입니다 (개인 지메일 계정이면 접근 거부됨)
3. 고객사별 응답 수, 평균 종합점수, 평균 재계약의향이 표로 나오고, 행을 클릭하면 개별 응답(강점/개선점)이 펼쳐집니다
4. 새 응답이 들어온 뒤에는 새로고침(F5)만 하면 최신 데이터로 반영됩니다 (별도 명령어 없음)

## 5. 문항/배점 수정 후 재배포하기

문항 문구나 배점 기준을 바꾸고 싶을 때는 저한테 "이 항목 문구/배점 바꿔줘"라고 말씀하시면 파일을 수정해 드립니다. 수정 후 **실제 응답 페이지에 반영하려면** 아래를 직접 하셔야 합니다 (구글 계정 작업):

1. [script.google.com](https://script.google.com)에서 프로젝트 열기
2. `Code.gs` / `Index.html` 내용을 최신 것으로 교체
3. "배포" → "배포 관리" → 기존 배포 옆 연필 아이콘 → 버전 "새 버전"으로 배포
4. 웹앱 URL은 그대로 유지되므로, 이미 보낸 링크는 다시 안 보내도 됨

> 참고: 마법사(`webapp/index.html`)와 실제 응답 페이지(`webapp/gsurvey/`)는 문항 데이터가 각각 따로 있어서, 마법사에서 바꾼 항목이 자동으로 응답 페이지에 반영되지 않습니다. 두 곳 다 반영해야 할 때는 "마법사랑 구글 설문 둘 다 반영해줘"라고 말씀해 주세요.

## 5. 코드를 GitHub에 올리기

**직접 안 하셔도 됩니다.** 저한테 "깃헙에 올려줘" 또는 "커밋하고 푸시해줘"라고 말씀하시면 아래 과정을 대신 실행합니다.

```bash
git add -A                    # 바뀐 파일 전부 담기
git commit -m "변경 내용 요약"   # 스냅샷 저장
git push                       # GitHub에 반영
```

**단, 최초 1회는 직접 인증**이 필요했습니다 (이미 완료하셨다면 이 부분은 넘어가도 됩니다):
```bash
cd "/Users/luna.lovegood/Desktop/AI Native/Customer Satisfaction "
git push -u origin main
```
비밀번호 자리에는 GitHub 비밀번호가 아니라 **토큰**을 입력합니다. 토큰 만드는 곳: https://github.com/settings/tokens (권한: `repo` 체크)

## 6. 온라인 주소(Vercel)로 배포하기

다른 사람이 URL로 마법사에 접속할 수 있게 하는 단계입니다. **계정 연결이라 최초 설정은 직접** 해주셔야 합니다.

1. https://vercel.com 접속 → "Continue with GitHub" 로그인
2. "Add New..." → "Project" → `nicejeantime/Ray` 저장소 선택 → "Import"
3. "Root Directory"를 `webapp`으로 변경 → "Deploy"
4. 완료되면 `https://ray-xxxx.vercel.app` 같은 주소 생성됨 — 이게 공개 URL

**이후 재배포는 자동입니다.** GitHub에 푸시(5번)만 하면 Vercel이 알아서 새 버전을 다시 배포합니다. 별도 명령어 없음.
