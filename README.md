# 디케이테크인 고객 만족도 조사 프로젝트

운영 서비스(챗봇/웹/CRM/앱)에 대한 고객사 만족도 조사를 설계·발행하고, 응답을 구글 시트로 모아 사내 대시보드로 취합해 보는 도구 모음입니다.

## 구성

| 경로 | 역할 |
|---|---|
| [webapp/index.html](webapp/index.html) | **설계 마법사** — 고객사 정보/서비스 형태/평가 항목/배점을 정하고, 응답 URL을 생성 (Vercel에 공개 배포) |
| [webapp/gsurvey/](webapp/gsurvey/설정가이드.md) | **응답 수집 웹앱 + 대시보드** — 고객사가 접속해 점수를 입력하는 설문 페이지, 그리고 사내 전용 결과 취합 대시보드 (Google Apps Script + 구글 시트) |
| [2026_고객사_만족도조사_설계안.md](2026_고객사_만족도조사_설계안.md) | 항목 설계 배경 리서치 (왜 이 항목들을 쓰는지) |

## 사용 흐름

```
1. webapp/gsurvey/ 를 Google Apps Script에 배포 (최초 1회, 설정가이드.md 참고)
   → 설문용 배포(누구나 링크 접속)와 대시보드용 배포(사내 계정만) 2개로 나눔
2. webapp/index.html 을 열어 고객사별로 조사 구조를 설계
3. 마지막 단계 "URL 생성"으로 고객사 전달용 링크 생성
4. 링크를 고객사에 전달 → 응답은 구글 시트에 자동 누적
5. 대시보드 URL(?view=dashboard)로 접속해 고객사별 종합 결과 확인
```

## 알아둘 점

`webapp/index.html`(설계 마법사)과 `webapp/gsurvey/Code.gs`(실제 응답 페이지)는 브라우저와 Google Apps Script라는 서로 다른 런타임에서 동작하는 별개의 파일이라, 평가 항목 데이터가 각자 따로 정의되어 있습니다. 마법사에서 항목을 껐다 켜거나 배점을 조정해도 실제 응답 페이지 문항에는 자동 반영되지 않습니다 — 응답 문항 자체를 바꾸려면 `gsurvey/Code.gs`의 `MODULES_DATA`를 직접 수정해야 합니다. 두 파일 모두 같은 서비스 형태(커넥트톡/웹/CRM/앱) 구조를 공유하므로 새 서비스 형태를 추가할 땐 양쪽에 동일하게 반영하세요.
