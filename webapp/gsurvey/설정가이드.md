# 커스텀 웹 설문 + 구글 시트 응답 수집 + 대시보드 — 배포 가이드

이 폴더(`gsurvey/`)는 **직접 디자인한 웹페이지**를 설문지로 쓰고, **제출된 응답은 구글 시트에 자동 저장**되며, **같은 데이터를 사내 대시보드로도** 볼 수 있는 구조입니다.

구성:
- `Code.gs` — 서버 로직 (페이지 렌더링 + 시트 저장 + 대시보드 데이터 집계)
- `Index.html` — 응답자(고객사)가 보는 실제 설문 페이지 (커스텀 디자인)
- `Dashboard.html` — 사내 전용 결과 취합 대시보드

설문 페이지는 "링크만 있으면 누구나" 접속해야 하고, 대시보드는 "회사 구글 계정만" 봐야 하므로 **배포를 2개로 나눕니다.**

---

## 1. 배포 방법 (최초 1회)

1. [script.google.com](https://script.google.com) → "새 프로젝트"
2. 기본 생성된 `Code.gs` 내용을 지우고, 이 폴더의 `Code.gs` 내용을 전체 붙여넣기
3. 좌측 "+" → "HTML" 선택 → 파일명을 정확히 `Index` 로 입력 (자동으로 `Index.html`이 됨)
4. 이 폴더의 `Index.html` 내용을 전체 붙여넣기
5. 좌측 "+" → "HTML" 선택 → 파일명을 정확히 `Dashboard` 로 입력
6. 이 폴더의 `Dashboard.html` 내용을 전체 붙여넣기
7. `Code.gs` 상단의 `DASHBOARD_DOMAIN` 값을 실제 회사 구글 워크스페이스 도메인으로 수정 (예: `'dktechin.com'`)

### 배포 A — 설문용 (고객사에게 보낼 링크)
"배포" > "새 배포"
- 유형 선택(톱니바퀴): **웹 앱**
- 설명: `설문용`
- 실행 계정: **나**
- 액세스 권한: **링크가 있는 모든 사용자**
- "배포" 클릭 → 처음이면 권한 승인 진행 → 나오는 URL을 "설문 URL"로 보관

### 배포 B — 대시보드용 (사내 전용 링크)
"배포" > "새 배포" (한 번 더, 별도 배포)
- 유형: **웹 앱**
- 설명: `대시보드용`
- 실행 계정: **나**
- 액세스 권한: **"○○○(회사 도메인) 내 모든 사용자"** (Google Workspace 조직 계정 기준. 이 옵션이 안 보이면 조직에서 Web App 배포를 제한한 것이니 관리자에게 확인)
- "배포" 클릭 → 나오는 URL을 "대시보드 URL"로 보관

두 URL 모두 `https://script.google.com/macros/s/XXXX/exec` 형태지만 서로 다른 주소입니다. 설문 URL은 4번처럼 파라미터를 붙여 쓰고, 대시보드 URL은 뒤에 `?view=dashboard`만 붙이면 됩니다.

---

## 2. 고객사별 링크 만들기

웹앱 URL 뒤에 쿼리 파라미터를 붙여서 고객사별로 다른 링크를 만듭니다.

```
{웹앱URL}?client=한국%20암웨이&type=chatbot
```

| 파라미터 | 설명 |
|---|---|
| `client` | 고객사명 (URL 인코딩 필요, 공백은 %20) — 응답 페이지 상단에 고정 표시됨 |
| `type` | 서비스 형태: `chatbot` / `web` / `crm` / `app`. 미리 알고 있으면 넣어서 선택 단계를 건너뛰게 함. 모르면 파라미터를 아예 빼면 응답자가 직접 고름 |

10개 고객사 예시:
```
{웹앱URL}?client=한국%20암웨이&type=chatbot
{웹앱URL}?client=고객사B&type=web
{웹앱URL}?client=고객사C&type=crm
...
```

> 이 링크들을 표로 정리해서 각 담당자에게 개별 전달하면 됩니다. (엑셀/시트에 고객사명-URL 매핑표를 만들어두면 관리가 편함)

---

## 3. 응답자가 보는 화면

1. 상단 배너에 고객사명 고정 표시 (링크에 포함된 값)
2. (type 파라미터가 없을 때만) 서비스 형태 선택
3. 항목별 1~5점 평가 + 영역 추가 의견
4. 종합의견(강점/개선점) + 재계약·추천 의향
5. 제출 → "제출 완료" 화면

---

## 4. 응답 확인 (구글 시트)

- 최초 응답이 제출되는 순간, "2026 운영서비스 만족도 조사_응답" 이라는 구글 시트가 자동 생성되고 이후 모든 응답이 여기에 한 행씩 쌓입니다.
- 시트 링크를 확인하려면: Apps Script 편집기에서 `getSheetUrl` 함수를 선택해 실행 → 실행 로그에서 URL 확인
- 컬럼 구성: 제출시각, 고객사명, 서비스형태, (서비스별 항목 점수 — 해당 안 되는 서비스 항목은 빈 칸), 강점, 개선점, 재계약/추천 의향

---

## 5. 대시보드 보기 (사내 전용)

1. "배포 B"에서 만든 **대시보드 URL** 뒤에 `?view=dashboard`를 붙여서 접속
   ```
   {대시보드용 배포 URL}?view=dashboard
   ```
2. 회사 구글 계정(`DASHBOARD_DOMAIN`에 설정한 도메인)으로 로그인되어 있어야 보입니다. 다른 도메인 계정이거나 비로그인 상태면 "접근 권한이 없습니다" 화면이 나옵니다.
3. 화면 구성: 고객사별 응답 수, 평균 종합점수(5점 만점), 평균 재계약/추천 의향, 최근 서비스 형태·평가기간을 표로 보여주고, 행을 클릭하면 해당 고객사의 개별 응답(강점/개선점 서술)이 펼쳐집니다.
4. 데이터는 4번의 구글 시트를 그대로 읽어서 매번 새로 계산하므로, 새 응답이 들어온 뒤 대시보드를 새로고침(F5)하면 바로 반영됩니다.

---

## 6. 문항 수정하고 싶을 때

`Code.gs`의 `MODULES_DATA` 객체에서 서비스 형태별(`chatbot`,`web`,`crm`,`app`) 항목의 `name`/`detail`을 수정하면 됩니다. 수정 후:
- 배포 > 배포 관리 > **배포 A, 배포 B 둘 다** 옆 연필 아이콘 > 버전을 "새 버전"으로 변경 후 배포해야 양쪽 URL에 반영됩니다 (URL 자체는 그대로 유지되므로 이미 보낸 링크는 재사용 가능).

---

## 7. 문제 해결

| 증상 | 원인/해결 |
|---|---|
| 로컬에서 Index.html을 직접 열면 제출이 안 됨 | 정상입니다. `google.script.run`은 Apps Script가 서빙하는 배포 URL에서만 동작합니다. 반드시 배포된 웹앱 URL로 테스트하세요. |
| 제출 시 오류 메시지 | Apps Script 편집기 좌측 "실행" 탭에서 최근 실행 로그를 확인하면 원인 파악 가능 |
| 응답 시트가 안 보임 | `getSheetUrl` 함수를 한 번 실행해서 로그의 링크를 확인 (최초 응답이 1건이라도 있어야 시트가 생성됨) |
| 대시보드에서 "접근 권한이 없습니다" | (1) 대시보드 URL이 아니라 설문 URL로 접속했거나 (2) 회사 도메인 계정으로 로그인하지 않았거나 (3) `DASHBOARD_DOMAIN` 값이 실제 도메인과 다름 — 셋 다 확인 |
| 대시보드에 "액세스 권한 내 모든 사용자" 옵션이 안 보임 | 조직의 Google Workspace 관리자가 Apps Script 배포 범위를 제한한 경우입니다. 관리자에게 문의하거나, 임시로 "나만" 권한으로 배포해서 본인만 사용 |
