# 디케이테크인 고객 만족도 조사 마법사 — 이용 매뉴얼

대상 파일: `index.html`
별도 설치나 서버 없이, 브라우저만 있으면 바로 사용할 수 있습니다.
이 도구는 평가표 구조를 잡는 **설계 도구**이며, 실제 고객사가 응답을 입력하는 화면은 [`gsurvey/`](gsurvey/설정가이드.md)가 담당합니다. 아래 4절 참고.

---

## 1. 실행 방법

- Finder에서 `index.html`을 더블클릭 → 기본 브라우저로 열립니다.
- 또는 터미널에서: `open "index.html"`

인터넷 연결이 없어도 동작합니다 (외부 서버/라이브러리를 쓰지 않는 순수 웹페이지).

---

## 2. 사용 순서 (5단계)

### 1단계 — 고객사 정보
| 입력 항목 | 설명 |
|---|---|
| 고객사명 * | 평가표 상단에 표시될 이름 (예: 디케이테크인) |
| 서비스명 * | 평가 대상 서비스명 (예: 커넥트톡) |
| 평가 기간 * | 시작일 / 종료일 |
| 평가자명 * | 고객사 담당자가 직접 입력 |
| 계약 * | 정액 운영 / 성과 기반(Outcome-based) / SLA 계약 — **복수 선택 가능**. 선택 결과는 4단계 배점 기본값에 영향 |

운영 시간은 24시간 365일 고정이라 별도 입력란이 없습니다. `*` 표시 항목은 모두 필수이며, 비어 있으면 다음 단계로 넘어가지 않고 안내 문구가 표시됩니다.

### 2단계 — 서비스 형태 선택
카드 중 하나를 클릭해서 선택합니다 (선택되면 파란 테두리로 표시).

| 카드 | 해당하는 경우 |
|---|---|
| 커넥트톡 | 챗봇, AI 상담 자동화 운영 서비스 |
| 웹 서비스 운영 | 웹사이트/웹서비스 운영·유지보수 |
| CRM 솔루션 | CRM 시스템 구축/운영 |
| 모바일 앱 | 모바일 앱 운영 |

> 서비스 형태를 다시 클릭해서 바꾸면, 3단계 항목 체크 상태가 초기화(기본값으로 재설정)됩니다.

### 3단계 — 평가 항목 선택
- 선택한 서비스 형태의 **생산성/품질/성과 전 항목**이 체크박스로 나타나며, 기본적으로 전부 체크되어 있습니다.
- 필요 없는 항목은 체크 해제하면 되고, 각 영역(생산성/품질/성과)마다 **최소 1개는 남겨야** 다음으로 진행됩니다.
- "보안 권장" 뱃지가 붙은 항목은 개인정보·민감정보를 다루는 서비스일 때 포함을 권장하는 항목입니다.
- 하단의 "재계약/추천 의향 문항 추가"는 종합의견에 정성적 신뢰도 질문을 하나 더 넣을지 여부입니다 (기본 켜짐, 권장).

### 4단계 — 배점 비중 조정
- 생산성(정량) / 품질(정성) / 성과 세 칸에 점수를 입력합니다. **세 칸의 합이 반드시 100이어야** 다음 버튼이 활성화됩니다.
- 합계가 100이 아니면 하단에 빨간 글씨로 표시됩니다.
- "○○ 계약 적용" 버튼을 누르면 1단계에서 고른 계약 기준 추천 비율로 자동 채워집니다.
  - 정액 운영: 40 / 30 / 30
  - 성과 기반: 35 / 25 / 40
  - SLA 계약: 45 / 25 / 30
  - 계약을 복수 선택했다면 성과 기반 > SLA > 정액 순으로 우선 적용됩니다.

### 5단계 — 결과 확인 및 URL 생성
완성된 평가표 구조 미리보기가 나타납니다. 상단 버튼:

| 버튼 | 동작 |
|---|---|
| 처음부터 다시 | 모든 입력값을 초기화하고 1단계부터 재시작 |
| 이전으로 | 4단계(배점 조정)로 돌아가기 |
| URL 생성 | 고객사에게 보낼 실제 응답 페이지 URL을 만듭니다 (아래 4절 참고) |

---

## 3. 자주 하는 조정

### 항목 문구나 배점 기준값을 고치고 싶을 때
`index.html`을 텍스트 에디터(VSCode 등)로 열어 `const MODULES = { ... }` 부분을 찾습니다. 서비스 형태별(`chatbot`, `web`, `crm`, `app`)로 항목이 나뉘어 있습니다.

```
{key:'firstResponse', name:'1차 대응 시간', def:'접수 ~ 최초 응답 소요 시간', sla:'2H 이내', weight:10}
```
- `key`: 항목 고유 식별자 (영문, 다른 항목과 겹치지 않게) — 3단계 체크 상태를 기억하는 데 사용됩니다
- `name`: 항목명, `def`/`detail`: 설명, `sla`: 목표 수준, `weight`: 해당 섹션 내 상대적 비중(점수는 4단계 합계에 맞춰 자동 배분됨)

수정 후 저장(Cmd+S)하고, 브라우저에서 새로고침(Cmd+R)하면 바로 반영됩니다.

### 새 서비스 형태를 추가하고 싶을 때
`MODULES` 안에 기존 서비스(예: `web`)를 통째로 복사해서 새 key로 추가하고, 항목만 바꾸면 2단계 카드 목록에 자동으로 나타납니다. 이때 `gsurvey/Code.gs`의 `MODULES_DATA`에도 동일한 서비스를 추가해야 실제 응답 페이지에도 반영됩니다 (두 파일은 서로 다른 런타임이라 자동 동기화되지 않습니다).

### 계약별 추천 배점을 바꾸고 싶을 때
`const CONTRACT_PRESETS = { ... }` 부분의 숫자를 수정합니다.

## 4. URL 생성이 하는 일 (gsurvey 연동)

5단계의 "URL 생성" 버튼은 문서를 만드는 게 아니라, **고객사가 실제로 접속해 응답을 입력할 웹 페이지 URL**을 만들어줍니다.

1. 최초 클릭 시 배포된 `gsurvey` 웹앱 URL을 한 번 입력받습니다 (브라우저에 저장되어 이후엔 다시 묻지 않음). 아직 배포 전이라면 [gsurvey/설정가이드.md](gsurvey/설정가이드.md)를 먼저 진행하세요.
2. 저장된 URL에 `?client=고객사명&type=서비스형태` 파라미터를 붙여 완성된 링크를 보여줍니다.
3. "복사" 버튼으로 복사해서 고객사 담당자에게 전달하면 됩니다.

> 주의: 현재 `gsurvey`는 서비스 형태별로 **고정된 문항 세트**를 사용합니다. 이 마법사의 3단계(항목 on/off)나 4단계(배점)에서 커스터마이징한 내용은 URL에 반영되지 않고, 실제 응답 페이지는 `gsurvey/Code.gs`에 정의된 기본 문항 그대로 나갑니다. 이 마법사는 "어떤 구조로 평가할지 설계·확인"하는 용도로, 실제 배포되는 문항을 바꾸려면 `gsurvey/Code.gs`를 직접 수정해야 합니다.

## 5. 문제 해결

| 증상 | 원인/해결 |
|---|---|
| "다음" 버튼이 안 눌림 | 필수 항목(고객사명·서비스명·평가기간·평가자명·계약)이 비어 있음 |
| 3단계에서 다음이 안 눌림 | 생산성/품질/성과 중 한 영역이 전부 체크 해제됨 — 최소 1개는 남겨야 함 |
| 4단계 다음 버튼이 비활성화 | 배점 합계가 100이 아님 |
| URL 생성이 안 됨 | `gsurvey` 웹앱이 아직 배포되지 않았거나, 저장된 URL이 잘못됨 → 결과 화면의 "여기"를 눌러 URL 재설정 |
| 수정한 내용이 반영 안 됨 | 파일 저장 후 브라우저 새로고침(Cmd+R) 필요 |
